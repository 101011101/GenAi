// How It Works

Red team the fraud space.
Before attackers do.

STEP 01 / ORCHESTRATOR

Decompose the fraud type

Claude Opus + extended thinking breaks a natural language fraud description into every variation dimension: hop count, timing, topology, evasion, extraction.

STEP 02 / PERSONAS

Generate criminal profiles

5 distinct criminal personas — different risk tolerance, resources, and geography. Each seeds structurally different fraud behavior across the variant space.

STEP 03 / CONSTRUCTORS

Parallel agent fleet

N FraudConstructor agents run in parallel, each assigned a coverage cell. 4-step LLM reasoning chain: persona analysis → network planning → participant profiles → transaction generation.

STEP 04 / CRITIC

Quality gate + revision

CriticAgent scores realism (≥7) and distinctiveness (≥6). Failed variants get specific feedback and are revised. Coverage matrix tracks saturation.

STEP 05 / OUTPUT

Labeled ML-ready dataset

CSV (XGBoost/RF), JSON (GNN), and graph adjacency list. Every row is labeled: is_fraud, fraud_role (placement / hop_N_of_M / extraction / cover_activity), variant metadata.

// Why Not Statistical Synthesis

Agents reason. Statistics mirror.

IBM AMLSIM / STATISTICAL TOOLS

More of what you already know

✗Generates examples around known distributions — noise on existing data
✗No evasion logic — mirrors patterns, doesn't reason about controls
✗Limited to parameterized model space — can't explore structurally novel variants
✗No persona-driven behavior — all variants converge toward "average fraud"
FRAUDGEN / ADVERSARIAL AGENTS

What you don't know yet

✓LLM agents reason from first principles — explore structurally different variants
✓Explicit evasion reasoning — agents think about what controls to evade and why
✓Coverage matrix guarantees diverse variant space, not clustering
✓Criminal personas produce structurally different outputs — not average fraud
