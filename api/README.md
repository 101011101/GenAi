# FraudGen API

FastAPI backend for the FraudGen pipeline. Wraps the existing orchestrator/sub-agent system and exposes the run lifecycle over HTTP so any frontend can consume it.

## Quick start

```bash
# From repo root
bash api/start.sh
```

Server starts on **http://localhost:8000**. Interactive docs at **http://localhost:8000/docs**.

**Requirements:** Python 3.10+, deps in `api/requirements.txt`, an `.env` file at repo root with `ANTHROPIC_API_KEY`.

```bash
pip install -r api/requirements.txt
```

### CORS

Set `ALLOWED_ORIGINS` in your `.env` to a comma-separated list of allowed origins. Defaults to `http://localhost:3000,http://localhost:3001,*`.

```
ALLOWED_ORIGINS=https://my-app.vercel.app,http://localhost:3000
```

## Endpoints

### POST `/api/runs`

Start a new pipeline run. Returns immediately with a `run_id` — the pipeline executes in a background thread.

**Request body:**
```json
{
  "fraud_description": "layered mule account network using ACH transfers",
  "variant_count": 7
}
```

**Response:**
```json
{ "run_id": "run_a3f9..." }
```

---

### GET `/api/runs/{run_id}/status`

Poll this during the run. Returns live state + logs.

**Response shape — `StatusData`:**
```json
{
  "run_id": "run_a3f9...",
  "phase": "generating",
  "variants_completed": 3,
  "variants_total": 7,
  "total_cost_usd": 0.42,
  "cost_cap_usd": 5.0,
  "start_time": 1710000000.0,
  "control_signal": "run",
  "errors": [],
  "event_log": ["[12:01:05] Orchestrator complete", "..."],
  "variant_log": [ /* see VariantSummary below */ ],
  "coverage_cells": [ /* see CellStatus below */ ]
}
```

**`phase` values:** `"idle"` → `"planning"` → `"generating"` → `"done"` | `"stopped"` | `"error"`

---

### POST `/api/runs/{run_id}/control`

Pause, resume, or stop a running pipeline.

**Request body:**
```json
{ "signal": "pause" }
```

**`signal` values:** `"run"` (resume), `"pause"`, `"stop"`

**Response:** `{ "ok": true, "signal": "pause" }`

---

### GET `/api/runs/{run_id}/matrix`

Coverage matrix — the dimensions the orchestrator decomposed and the status of each cell.

**Response:**
```json
{
  "dimensions": [
    { "name": "hop_count", "values": [3, 5, 7] },
    { "name": "timing", "values": ["burst", "spread", "random"] }
  ],
  "cells": [ /* see CellStatus below */ ]
}
```

---

### GET `/api/runs/{run_id}/personas`

Fraud personas generated by the orchestrator.

**Response:**
```json
{
  "personas": [
    {
      "persona_id": "p_001",
      "name": "Marcus Webb",
      "risk_tolerance": "high",
      "backstory": "Former bank employee...",
      "evasion_targets": ["velocity checks", "AML rules"],
      "geographic_scope": "domestic"
    }
  ]
}
```

---

### GET `/api/runs/{run_id}/results`

Full results once the run completes (or at any point for partial results).

**Response:**
```json
{
  "variants": [ /* array of ScoredVariant objects */ ],
  "stats": {
    "variants_generated": 7,
    "mean_realism": 0.84,
    "mean_distinctiveness": 0.76,
    "mean_critic_score": 0.80,
    "coverage_pct": 71.4,
    "total_transactions": 238
  }
}
```

---

### GET `/api/runs/{run_id}/dataset?page=1&page_size=100`

Paginated transaction-level dataset (the flat CSV, parsed to JSON rows).

**Response:**
```json
{
  "rows": [
    {
      "transaction_id": "txn_001",
      "timestamp": "2024-03-15T10:23:00",
      "amount": "4500.00",
      "channel": "ACH",
      "is_fraud": "true",
      "fraud_role": "mule_hop",
      "variant_id": "var_001",
      "persona_name": "Marcus Webb"
    }
  ],
  "total": 238,
  "page": 1,
  "page_size": 100
}
```

---

### GET `/api/runs/{run_id}/export/{format}`

Download a completed run's output files.

| `format` | File | MIME type |
|---|---|---|
| `csv` | `dataset.csv` | `text/csv` |
| `json` | `dataset.json` | `application/json` |
| `graph` | `graph_adjacency.json` | `application/json` |
| `summary` | `run_summary.json` | `application/json` |

Returns a file download. 404 if the run isn't finished yet.

## Key data shapes

### `VariantSummary` (in `status.variant_log[]`)

```json
{
  "variant_id": "var_001",
  "status": "approved",
  "critic_score": 0.82,
  "realism_score": 0.84,
  "distinctiveness_score": 0.79,
  "attempt_count": 1
}
```

### `CellStatus` (in `status.coverage_cells[]`)

```json
{
  "cell_id": "cell_3_burst",
  "dimension_values": { "hop_count": 3, "timing": "burst" },
  "assigned_persona_id": "p_001",
  "assigned_persona_name": "Marcus Webb",
  "status": "completed",
  "attempt_count": 1,
  "critic_score": 0.82,
  "variant_id": "var_001"
}
```

**`status` values:** `"empty"` → `"assigned"` → `"in_progress"` → `"completed"` | `"failed"` | `"reassigned"`

### `StatusData` phases

```
POST /api/runs
       ↓
   idle → planning → generating → done
                                → stopped  (manual stop)
                                → error    (unrecoverable)
```

Poll `GET /status` every 1–2s while `phase` is `planning` or `generating`.

---

## Typical frontend flow

```
1. POST /api/runs                    → get run_id
2. Poll GET /api/runs/{id}/status    → watch phase, variant_log, coverage_cells
3. Once phase == "done":
   GET /api/runs/{id}/results        → summary stats + scored variants
   GET /api/runs/{id}/dataset        → paginated transaction rows
   GET /api/runs/{id}/export/csv     → download file
```
